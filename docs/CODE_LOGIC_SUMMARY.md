## 核心模块

### 1. 核心基础设施 (core/)

#### 1.1 EventLoop (事件循环)
**文件**: `include/core/event_loop.h`, `src/core/event_loop.cpp`

**职责**:
- 主事件循环，负责调度和协调所有模块
- 处理系统事件（触摸、帧结束等）
- 管理频率更新和策略执行
- 空闲状态检测和频率下探

**主要流程**:
```
1. 初始化
   - 创建 epoll 实例
   - 创建定时器
   - 初始化所有子系统
   - 构建 CPU-domain 映射

2. 主循环
   - 等待事件（epoll_wait）
   - 处理定时器事件（100ms 周期）
   - 收集系统数据
   - 处理负载特征
   - 执行调度决策
   - 应用频率调整

3. 空闲状态检测
   - 检测整机负载 < 5%
   - 检测无触摸 2 分钟
   - 逐步下探到最低频率
   - 违反条件时立即退出

4. 频率更新
   - Rate limiting（限速）
   - 游戏场景：50ms 间隔
   - 日常场景：100ms 间隔
   - 避免频繁调频
```

**关键优化**:
- CPU-domain 映射（O(n) → O(1)）
- 文件描述符缓存
- 逐步下探策略
- 动态限速间隔

### 2. 设备相关 (device/)

#### 2.1 MigrationEngine (迁移引擎)
**文件**: `include/device/migration_engine.h`, `src/device/migration_engine.cpp`

**职责**:
- 核心迁移决策
- 动态负载均衡
- 任务分类
- 功率感知调度

**主要流程**:
```
1. 初始化
   - 检测设备代数
   - 加载 SoC 配置
   - 初始化负载历史
   - 初始化冷却期

2. 更新负载
   - 更新所有核心的负载
   - 更新运行队列
   - 更新唤醒次数
   - 计算负载趋势

3. 迁移决策
   a. 老旧设备（865及以前）
      - 小核→中核：负载 > 25%
      - 中核→小核：负载 < 23.5%
      - 中核→大核：负载 > 62.5%
      - 小核过载保护：负载 > 75% 或运行队列 >= 4

   b. 现代设备（8+及以后）
      - 小核→中核：负载 > 37.5%
      - 中核→小核：负载 < 12.5%
      - 中核→大核：负载 > 50%
      - 小核过载保护：负载 > 75% 或运行队列 >= 4

   c. 全大核设备（8 Elite, 9400等）
      - 动态负载均衡
      - 超大核优先
      - 性能核次之
      - 低负载时均匀分布

4. 动态调整
   - 根据温度调整阈值
   - 根据电池调整阈值
   - 根据场景调整阈值
   - 根据功耗调整阈值
   - 根据中核利用率调整阈值
   - 根据小核最大负载调整阈值

5. 任务分类
   - 计算密集型：负载 > 50% → 大核
   - 内存密集型：负载 < 50% 且唤醒次数 > 100 → 中核
   - IO 密集型：运行队列 > 2 → 小核
   - 未知任务：中核（默认）

6. 目标选择
   - 综合评分：负载 + 运行队列 + 负载趋势
   - 负载均衡：优先选择负载均衡
   - 功率感知：功耗高时减少迁移
   - 冷却期：避免频繁迁移
```

**关键优化**:
- 动态负载均衡
- 小核过载保护
- 任务分类
- 功率感知调度
- 动态阈值调整

## 总结

HyperPredict 是一个高性能 Android CPU 调度优化系统，通过以下机制实现性能和功耗的平衡：

1. **多时间尺度 EMA** - 精准预测负载变化
2. **动态负载均衡** - 自动适应负载变化
3. **任务分类** - 根据任务类型选择最优核心
4. **功率感知调度** - 根据功耗动态调整策略
5. **空闲状态检测** - 空闲时下探到最低频率
6. **小核过载保护** - 防止小核负载拉满
7. **趋势预测** - 提前升频，延迟降频
8. **温控缩放** - 根据温度动态调整频率
9. **UCLamp 设置** - 精确控制 CPU 性能
10. **Rate limiting** - 避免频繁调频

通过这些优化，HyperPredict 在保持性能的同时，显著降低了功耗，提高了能效比。
